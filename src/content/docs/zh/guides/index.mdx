当你想直接告诉 AI 代理该做什么，而不是手动拼接底层步骤时，就用这些指南。

## 从合适的指南开始

<CardGrid stagger>
  <LinkCard title="快速开始" href="/zh/getting-started/">
    连接代理、创建第一个项目、加入跟踪器，并确认第一条分析闭环已经跑通。
  </LinkCard>
  <LinkCard title="为你的 Paperclip 公司设置 Agent Analytics" href="/zh/guides/paperclip/">
    在 Paperclip 里先给 CEO 创建一个任务，完成代理登录批准交接，并确认第一条真实分析读取已经跑通。
  </LinkCard>
  <LinkCard title="5 分钟完成第一个项目" href="/zh/guides/first-project-in-5-minutes/">
    从代理已接入，走到项目创建、tracker 放好、第一个 page view 验证通过，以及第一个真实分析问题答出来。
  </LinkCard>
  <LinkCard title="SPA 与虚拟页面跟踪" href="/zh/guides/spa-and-virtual-page-tracking/">
    在 SPA 中优先使用 URL 驱动的路由，并只在必要时手动发送虚拟 page view，这样页面跟踪才不会重复计数。
  </LinkCard>
  <LinkCard title="AI 代理实验跟踪" href="/zh/guides/ai-agent-experiment-tracking/">
    通过让代理处理实验工作流，来规划、上线、QA 并读取浏览器侧 A/B 测试结果。
  </LinkCard>
</CardGrid>

如果你是从 Paperclip 公司的工作流进来的，请先看[为你的 Paperclip 公司设置 Agent Analytics](/zh/guides/paperclip/)。如果你需要按环境进行安装，请使用[安装中心](/zh/installation/)。如果你需要浏览器侧或端点级的低层细节，请继续查看 [Tracker.js](/zh/reference/tracker-js/) 或 [API 参考](/zh/api/)。
