השתמשו במדריכים האלה כשאתם רוצים להגיד לסוכן ה-AI מה לעשות, במקום לחבר ידנית את כל השלבים ברמה הנמוכה.

## התחילו עם המדריך המתאים

<CardGrid stagger>
  <LinkCard title="התחלה מהירה" href="/he/getting-started/">
    חברו את הסוכן, צרו את הפרויקט הראשון, הוסיפו את הטרקר, וודאו שלולאת האנליטיקה הראשונה עובדת.
  </LinkCard>
  <LinkCard title="הגדירו את Agent Analytics עבור חברת ה-Paperclip שלכם" href="/he/guides/paperclip/">
    צרו משימה אחת עבור ה-CEO ב-Paperclip, השלימו את מסירת אישור ההתחברות לסוכן, וודאו שהקריאה האנליטית האמיתית הראשונה עובדת.
  </LinkCard>
  <LinkCard title="הפרויקט הראשון ב-5 דקות" href="/he/guides/first-project-in-5-minutes/">
    עברו מסוכן שכבר מחובר אל פרויקט שנוצר, טרקר שמותקן, page view ראשון שאומת ושאלת אנליטיקה אמיתית אחת שנענתה.
  </LinkCard>
  <LinkCard title="מעקב SPA ועמודים וירטואליים" href="/he/guides/spa-and-virtual-page-tracking/">
    שמרו על page tracking מדויק ב-SPA על ידי הסתמכות על ניתוב מבוסס URL כשאפשר, ועל page views וירטואליים ידניים רק כשצריך.
  </LinkCard>
  <LinkCard title="מעקב ניסויים לסוכני AI" href="/he/guides/ai-agent-experiment-tracking/">
    תכננו, השיקו, בצעו QA וקראו תוצאות של ניסויי A/B בדפדפן על ידי בקשה מהסוכן שלכם לנהל את ה-workflow.
  </LinkCard>
</CardGrid>

אם הגעתם מ-workflow של חברת Paperclip, התחילו עם [הגדירו את Agent Analytics עבור חברת ה-Paperclip שלכם](/he/guides/paperclip/). אם אתם צריכים הגדרה לפי סביבה, השתמשו ב-[מרכז ההתקנה](/he/installation/). אם אתם צריכים פירוט ברמת הדפדפן או ה-endpoint, המשיכו אל [Tracker.js](/he/reference/tracker-js/) או אל [עיון ב-API](/he/api/).
