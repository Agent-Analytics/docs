השתמשו במדריך הזה כשההתקנה של הסוכן כבר מאחוריכם ואתם רוצים את המסלול הכי ברור ללולאת ההצלחה הראשונה שלכם.

אם אתם עדיין צריכים קודם לחבר את הסוכן, חזרו אל [מרכז ההתקנה](/he/installation/) או אל הסקירה הקצרה של [התחלה מהירה](/he/getting-started/).

## 1. ודאו שהתקנת הסוכן כבר עובדת

בקשו מהסוכן:

```text
הצג את כל פרויקטי Agent Analytics שלי ואשר שאתה יכול לגשת לחשבון שלי.
```

אם הסוכן עדיין לא רואה את החשבון שלכם, עצרו כאן והשלימו קודם את [מסלול ההתקנה](/he/installation/) המתאים.

## 2. צרו את הפרויקט הראשון וקבלו את סניפט הטרקר

בקשו מהסוכן:

```text
צור פרויקט בשם my-site.com ותן לי את סניפט המעקב.
```

בשלב הזה אתם רוצים אחד משני מצבים:

- הסוכן יוצר את הפרויקט ומחזיר לכם סניפט טרקר להדבקה ידנית
- הסוכן יוצר את הפרויקט ויכול גם לשתול את הסניפט בעצמו

## 3. בחרו את המסלול שמתאים להגדרה שלכם

### הסוכן יכול לערוך קוד

אם לסוכן יש גישת כתיבה לאתר, בקשו ממנו:

```text
צור פרויקט בשם my-site.com, הוסף לאתר את סניפט המעקב של Agent Analytics, וכתוב לי בדיוק איזה קובץ שינית.
```

אחר כך בקשו:

```text
הראה לי איפה מיקמת את הסניפט ואשר שהוא ייטען בעמוד הראשון שאני רוצה לעקוב אחריו.
```

אם האתר שלכם משתמש ב-Astro, אמרו לסוכן להוסיף `is:inline` לתגית ה-`<script>` של הטרקר.

### הדבקה ידנית

אם הסוכן לא יכול לערוך את האתר, בקשו ממנו:

```text
צור פרויקט בשם my-site.com ותן לי את סניפט הטרקר המדויק שאני צריך להדביק לפני </body>.
```

הדביקו את הסניפט שהוחזר באתר, בצעו deploy אם צריך, ואז המשיכו.

אם האתר שלכם משתמש ב-Astro, הוסיפו `is:inline` לתגית ה-`<script>` של הטרקר לפני הפרסום.

## 4. טענו עמוד אמיתי וודאו את ה-`page_view` הראשון

פתחו בדפדפן עמוד שנמצא תחת מעקב באתר שלכם, ואז בקשו מהסוכן:

```text
בדוק האם my-site.com כבר קיבל את אירוע ה-page_view הראשון שלו היום, ותגיד לי איזה עמוד אתה רואה.
```

אם אתם רוצים בדיקה מפורשת עוד יותר, בקשו:

```text
ודא שהטרקר חי עבור my-site.com ואשר שה-page_view הראשון כבר נרשם.
```

אם אתם צריכים בדיקת sanity ידנית, אפשר גם לפתוח את [app.agentanalytics.sh](https://app.agentanalytics.sh) כדי לוודא שהפרויקט קיים ושהסניפט תואם למה שהסוכן השתמש בו.

## 5. שאלו שאלה אמיתית אחת על האנליטיקה

ברגע שה-`page_view` הראשון חי, שאלו שאלה פשוטה אחת:

```text
איך my-site.com מתקדם היום?
```

חלופות טובות:

- `מהם העמודים המובילים של my-site.com היום?`
- `הצג לי את תעבורת הבוטים של my-site.com היום.`

המטרה כאן פשוטה: להוכיח שהסוכן שלכם יכול ליצור את הפרויקט, לחווט את המעקב, לראות נתונים חיים ולענות על שאלה אמיתית בלי שתכתבו בקשות HTTP ידנית.

## השלב הבא

- השתמשו ב-[התחלה מהירה](/he/getting-started/) בשביל הסקירה הקצרה של כל מסלול ההגדרה.
- השתמשו ב-[התקנה](/he/installation/) אם החיבור של הסוכן עדיין צריך עבודה.
- השתמשו ב-[Tracker.js](/he/reference/tracker-js/) עבור אירועים דקלרטיביים, מעקב SPA, consent, קליקים, שגיאות ואפשרויות דפדפן נוספות.
- השתמשו ב-[מעקב ניסויים לסוכני AI](/he/guides/ai-agent-experiment-tracking/) אחרי שהפרויקט הראשון כבר חי, כשאתם רוצים שהסוכן ישיק ויקרא עבורכם ניסויי A/B בדפדפן.
- השתמשו ב-[API Reference](/he/api/) רק כשאתם צריכים פירוט raw ברמת endpoint.
